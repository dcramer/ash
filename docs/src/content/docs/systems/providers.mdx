Providers are transport layers between users and the agent runtime.

## Telegram Configuration

```toml
[telegram]
bot_token = "123456789:ABCdef..."
allowed_users = ["@yourusername", "123456789"]
allowed_groups = ["-100123456789"]
group_mode = "mention"

[telegram.passive]
enabled = false
```

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `bot_token` | string | `null` | Bot token from BotFather |
| `allowed_users` | list[string] | `[]` | Authorized users |
| `allowed_groups` | list[string] | `[]` | Authorized groups |
| `group_mode` | `mention`/`always` | `mention` | Group reply mode |
| `passive.enabled` | bool | `false` | Passive group listening |

## Runtime Notes

- `ash serve` starts the provider runtime and server
- Telegram provider supports active and passive group handling
- Access is controlled by `allowed_users`/`allowed_groups`

## Related Commands

```bash
uv run ash serve
uv run ash logs --component providers
uv run ash doctor
```

<Aside type="tip">
  Keep `allowed_users` configured for production bot deployments.
</Aside>
