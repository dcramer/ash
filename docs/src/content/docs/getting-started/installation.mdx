## Requirements

| Requirement | Version | Notes |
|-------------|---------|-------|
| Python | 3.12+ | Required |
| Docker | 20.10+ | For sandbox execution |
| uv | Latest | Recommended package manager |

## API Keys

Ash requires two API keys for the full experience:

| Provider | Purpose | Get a key |
|----------|---------|-----------|
| **Anthropic** | LLM (Claude powers the chat) | [console.anthropic.com](https://console.anthropic.com/) |
| **OpenAI** | Memory embeddings (semantic search) | [platform.openai.com](https://platform.openai.com/) |

<Aside type="tip">
  The setup wizard (`ash setup`) will prompt you for these keys and store them in your config file. No need to export environment variables.
</Aside>

## Install from Source

<Steps>

1. **Clone the repository**

   ```bash
   git clone https://github.com/dcramer/ash.git
   cd ash
   ```

2. **Install with uv**

   ```bash
   uv sync --all-groups
   ```

3. **Verify installation**

   ```bash
   uv run ash --help
   ```

</Steps>

## Configuration

Run the setup wizard:

```bash
uv run ash setup
```

This interactive wizard configures:
- API keys (Anthropic, OpenAI)
- Model preferences (Haiku for fast tasks, Sonnet for complex ones)
- Optional integrations (Telegram, Brave Search)

Configuration is saved to `~/.ash/config.toml`.

### Manual Configuration

You can also create the config file manually:

```toml
[anthropic]
api_key = "sk-ant-..."

[openai]
api_key = "sk-..."

[models.default]
provider = "anthropic"
model = "claude-haiku-4-5-20251001"

[models.sonnet]
provider = "anthropic"
model = "claude-sonnet-4-5-20250929"

[embeddings]
provider = "openai"
model = "text-embedding-3-small"
```

See [Configuration Overview](/configuration/overview/) for all options.

## Docker Sandbox

The sandbox isolates command execution. Build the image:

```bash
uv run ash sandbox build
```

Check status:

```bash
uv run ash sandbox status
```

## Database Setup

Initialize the SQLite database:

```bash
uv run ash upgrade
```

This runs migrations and verifies everything is ready.

## Workspace

Ash uses a workspace directory for personality:

```
~/.ash/workspace/
├── SOUL.md    # Assistant personality
└── USER.md    # User profile (optional)
```

The setup wizard creates these. Edit `SOUL.md` to customize how Ash behaves.

## Upgrading

```bash
cd ash
git pull
uv sync --all-groups
uv run ash upgrade
```

## Troubleshooting

### Docker not found

```bash
docker info
```

If this fails, install Docker and ensure the daemon is running.

### Permission denied

Add your user to the docker group:

```bash
sudo usermod -aG docker $USER
```

Log out and back in.

### Config issues

Check your config:

```bash
uv run ash config show
```

Validate it:

```bash
uv run ash config validate
```
