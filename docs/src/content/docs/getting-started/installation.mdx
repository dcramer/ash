## Requirements

| Requirement | Version | Notes |
|-------------|---------|-------|
| Python | 3.12+ | Required |
| Docker | 20.10+ | For sandbox execution |
| uv | Latest | Recommended package manager |

## API Keys

Ash requires an OpenAI API key for the full experience:

| Provider | Purpose | Get a key |
|----------|---------|-----------|
| **OpenAI** | LLM (GPT-5 powers the chat) + Memory embeddings | [platform.openai.com](https://platform.openai.com/) |

<Aside type="tip">
  Store your key in your `~/.ash/config.toml` file. You can also export it as `OPENAI_API_KEY` environment variable.
</Aside>

## Install from Source

<Steps>

1. **Clone the repository**

   ```bash
   git clone https://github.com/dcramer/ash.git
   cd ash
   ```

2. **Install with uv**

   ```bash
   uv sync --all-groups
   ```

3. **Verify installation**

   ```bash
   uv run ash --help
   ```

</Steps>

## Configuration

Create your configuration file:

```bash
uv run ash init
```

This creates `~/.ash/config.toml` with a template configuration. Edit it to add your API key:

```toml
[openai]
api_key = "sk-..."

[models.default]
provider = "openai"
model = "gpt-5.2"

[models.mini]
provider = "openai"
model = "gpt-5-mini"

[embeddings]
provider = "openai"
model = "text-embedding-3-small"
```

See [Configuration Overview](/configuration/overview/) for all options.

## Docker Sandbox

The sandbox isolates command execution. Build the image:

```bash
uv run ash sandbox build
```

Check status:

```bash
uv run ash sandbox status
```

## Database Setup

Initialize the SQLite database:

```bash
uv run ash upgrade
```

This runs migrations and verifies everything is ready.

## Workspace

Ash uses a workspace directory for personality:

```
~/.ash/workspace/
├── SOUL.md    # Assistant personality
├── USER.md    # User profile (optional)
└── skills/    # Custom skills
```

These are created automatically on first run. Edit `SOUL.md` to customize how Ash behaves.

## Upgrading

```bash
cd ash
git pull
uv sync --all-groups
uv run ash upgrade
```

## Troubleshooting

### Docker not found

```bash
docker info
```

If this fails, install Docker and ensure the daemon is running.

### Permission denied

Add your user to the docker group:

```bash
sudo usermod -aG docker $USER
```

Log out and back in.

### Config issues

Check your config:

```bash
uv run ash config show
```

Validate it:

```bash
uv run ash config validate
```
