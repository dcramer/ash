Manage the Docker sandbox for secure command execution.

## Usage

```bash
ash sandbox COMMAND [OPTIONS]
```

## Commands

### build

Build the sandbox Docker image:

```bash
ash sandbox build
```

Options:
- `--force`, `-f`: Rebuild even if image exists

### status

Check sandbox status:

```bash
ash sandbox status
```

Shows:
- Docker availability
- Image status
- Running containers

### clean

Remove stopped containers:

```bash
ash sandbox clean
```

Options:
- `--force`, `-f`: Also remove the sandbox image

## Examples

Initial setup:

```bash
ash sandbox build
ash sandbox status
```

Clean up containers:

```bash
ash sandbox clean
```

Full cleanup including image:

```bash
ash sandbox clean --force
```

## Verification

For security verification, use pytest:

```bash
uv run pytest tests/test_sandbox_verify.py -v
```

For manual testing prompts, use the `/test-sandbox` skill.

## Troubleshooting

### Build fails

Ensure Docker is running:

```bash
docker info
```

### Container issues

Check configuration:

```bash
ash config show
```

Review the `[sandbox]` section for issues.
