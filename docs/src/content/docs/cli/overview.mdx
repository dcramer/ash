Ash CLI command surface (`uv run ash --help`):

## Usage

```bash
uv run ash [OPTIONS] COMMAND [ARGS]...
```

## Top-Level Commands

| Command | Description |
|---------|-------------|
| [`auth`](/cli/auth/) | OAuth authentication |
| [`chat`](/cli/chat/) | Interactive chat or single prompt |
| [`config`](/cli/config/) | Configuration management |
| [`doctor`](/cli/doctor/) | Read-only operational checks |
| [`graph`](/cli/graph/) | Graph inspection utilities |
| [`init`](/cli/init/) | Create starter configuration |
| [`logs`](/cli/logs/) | View/search logs |
| [`memory`](/cli/memory/) | Memory management |
| [`people`](/cli/people/) | Person record management |
| [`sandbox`](/cli/sandbox/) | Sandbox management |
| [`schedule`](/cli/schedule/) | Scheduled task management |
| [`serve`](/cli/serve/) | Start server |
| [`service`](/cli/service/) | Background service management |
| [`sessions`](/cli/sessions/) | Session inspection/debugging |
| [`skill`](/cli/skill/) | Skill management |
| [`stats`](/cli/stats/) | Operational stats |
| [`upgrade`](/cli/upgrade/) | Upgrade checks/build |

## Key Environment Variables

| Variable | Description |
|----------|-------------|
| `ASH_HOME` | Base state directory (`config.toml`, logs, sessions, auth) |
| `OPENAI_API_KEY` | OpenAI API key |
| `ANTHROPIC_API_KEY` | Anthropic API key |
| `TELEGRAM_BOT_TOKEN` | Telegram bot token |
| `BRAVE_SEARCH_API_KEY` | Brave Search API key |
| `SENTRY_DSN` | Sentry DSN |

## Bootstrap Sequence

```bash
uv run ash init
uv run ash auth login
uv run ash config validate
uv run ash upgrade
uv run ash chat
```
